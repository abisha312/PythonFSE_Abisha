N+1 Query Problem
Observed the N+1 problem while loading enrollment, student, and course data.
Enabled SQL logging using echo=True to view generated SQL queries.
Fixing N+1 with joinedload()

Used eager loading:

session.query(Enrollment).options(
    joinedload(Enrollment.student),
    joinedload(Enrollment.course)
).all()

This reduced database calls and improved performance.

Django ORM (Bonus)
Created Django models for Department, Student, Course, and Enrollment.
Used:
Enrollment.objects.select_related('student', 'course').all()

to fetch related data efficiently and avoid the N+1 problem.
